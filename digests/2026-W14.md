# Weekly Digest: March 23–30, 2026

## Executive summary

This week’s announcements centered on two big themes: **agentic development workflows** and **production-ready platform operations for AI and cloud-native apps**. GitHub expanded Copilot and Actions with more autonomous capabilities, stronger security posture, and broader application security coverage, while Microsoft’s developer stack pushed local AI, Foundry-based agents, and Aspire deeper into day-to-day engineering workflows. On Azure, the most notable updates were in **AKS, observability, messaging, and integration**, with a steady stream of Kubernetes networking, storage, and monitoring enhancements. Security and governance also remained prominent, especially around **identity, AI guardrails, supply chain defense, and operational resilience**.

## Copilot & Agents

Microsoft 365 Copilot updates focused on making Copilot more practical in everyday work: better notebooks, smarter declarative agents, and more connectors and learning content. The overall direction is clear: richer collaboration and more capable task-specific agents built on newer model capabilities.

- Copilot Notebooks adds enhancements for creation, collaboration, and learning.
- Microsoft Copilot Connector v2 for Veeva Vault expands enterprise data integration.
- Microsoft 365 Copilot app learning series highlights real-world workday scenarios.
- Microsoft 365 Copilot Declarative Agents are getting smarter with GPT-5.2.
- Microsoft recapped Frontier Transformation and Wave 3 Microsoft 365 Copilot announcements.

## GitHub

GitHub’s week was defined by **more agentic Copilot experiences**, improved workflow visibility, and a strong security and trust narrative across Actions, privacy, and application security. For engineering teams, the most important changes were around pull request workflows, credential response, hosted runner customization, and AI-powered detections.

- Ask @copilot can now help resolve merge conflicts on pull requests.
- A new pull requests dashboard entered public preview for workflow visibility.
- Agent activity is now surfaced in GitHub Issues and Projects.
- Actions run summaries can now show Agentic Workflow configs.
- Custom images for GitHub-hosted runners are now generally available.
- Credential revocation API now supports GitHub OAuth and GitHub App credentials.
- GitHub outlined its GitHub Actions 2026 security roadmap.
- GitHub expanded application security coverage with AI-powered detections.
- Copilot usage metrics now identify active Copilot coding agent users.
- GitHub updated Copilot interaction data usage policy, Privacy Statement, and Terms of Service.

## Data & AI

The Data & AI announcements emphasized **lower-latency AI systems, better document extraction quality, and more operational control for AI workloads**. SQL tooling also improved in VS Code, reinforcing Microsoft’s push to bring data engineering and database development closer to modern developer workflows.

- Azure Managed Redis with Azure Databricks enables real-time feature serving for low-latency decisions.
- Azure Content Understanding focuses on improving extraction quality in Document AI.
- Microsoft Foundry introduced Priority Processing for performance-sensitive AI workloads.
- SQL code analysis in VS Code now supports configuring rules without editing project files.
- SQL schema management in VS Code adds a publish dialog and item templates.

## Developer Tools

This was a strong week for developer productivity, with updates spanning **VS Code, Aspire, Foundry Local, azd, the Azure SDK, and .NET learning content**. The common thread was faster paths from local experimentation to cloud deployment, especially for AI apps and agents.

- Visual Studio Code 1.114 shipped, alongside the earlier 1.113 release.
- Aspire 13.2 was announced, including TypeScript AppHost support.
- Foundry Local scenarios showed how to build local RAG and offline AI apps.
- Microsoft Foundry Labs was positioned as a fast lane from research to developer workflows.
- Azure SDK Release for March 2026 delivered the latest client library updates.

- A new vectorless reasoning-based RAG approach was introduced.
- azd can now help deploy an AI agent to Microsoft Foundry in minutes.
- Generative AI for Beginners .NET updated to version 2 on .NET 10.
- The .NET team shared lessons from ten months with Copilot Coding Agent in dotnet/runtime.

Azure platform announcements were heavy on **agentic operations, app modernization, messaging, and cloud-native deployment patterns**. - Aspire on Azure App Service is now generally available.

## Azure Infrastructure

Infrastructure coverage focused on **AI security guardrails, cost control, CLI-based automation, and private networking patterns**. The takeaway for platform teams is more prescriptive guidance for securing enterprise AI and operating Azure infrastructure safely at scale.

- Azure Storage synchronization across isolated private endpoint networks was detailed.
- Guardrails for generative AI were framed around securing developer workflows.
- GitHub Copilot CLI was highlighted for infrastructure deployment assistance.
- Enterprise AI security guidance covered red teaming, risk cards, and Azure guardrails.

## Security

Security updates concentrated on **identity, high-value asset protection, AI governance, and supply chain defense**. The most actionable items for defenders were the Trivy compromise guidance, Sentinel RBAC improvements, and concrete examples of Defender stopping advanced attack paths.

- Microsoft detailed how Defender protects high-value assets in real-world attack scenarios.
- Identity security was highlighted as the new pressure point in modern cyberattacks.
- Microsoft published guidance for detecting and defending against the Trivy supply chain compromise.
- Governing AI agent behavior was framed around aligning user, developer, role, and organizational intent.
- Microsoft Sentinel now supports Unified RBAC with row-level access.
- A Defender case study showed predictive shielding stopping GPO-based ransomware.
- Azure DDoS Protection status assessment guidance helps evaluate coverage across environments.

## Governance & Operations

Operations news emphasized **self-healing infrastructure, observability migration, and post-migration database maintenance modernization**. These are practical updates for teams focused on reliability and operational maturity after cloud migration.

- Azure VMware Solution private clouds now feature autonomous self-healing guidance.
- Dashboards can be copied from Dashboards with Grafana to Azure Managed Grafana.
- Database maintenance practices were redefined after moving from Db2 on mainframe to Azure SQL DB Hyperscale.

## Azure Updates

Azure Updates were dominated by **AKS and container platform enhancements**, plus a steady set of Azure SQL and PostgreSQL database improvements. For platform architects, the most important signals were around Kubernetes networking, observability, storage integration, and migration capabilities.

- AKS introduced blue-green agent pool upgrade in public preview.
- AKS added managed GPU metrics in Azure Monitor in public preview.
- Cross-cluster networking in Azure Kubernetes Fleet Manager entered public preview.
- Application routing with meshless Istio in AKS entered public preview.
- Microsoft Azure Kubernetes Application Network entered public preview.
- AI Agent for container networking troubleshooting entered public preview.
- Azure Container Storage v2.1.0 is now generally available with Elastic SAN integration and on-demand installation.
- Container network logs and container network metrics filtering for AKS are now generally available.
- Azure Monitor added Prometheus community recommended alerts for Azure Arc-enabled Kubernetes.
- Azure SQL, Cosmos DB mirroring, PostgreSQL migration, and SQL Managed Instance change event streaming all saw notable launches.

## Ecosystem

The ecosystem updates were smaller in volume but relevant across multi-cloud and infrastructure workflows. They touched infrastructure tooling, Oracle database options on Azure, and geospatial intelligence for energy decision-making.

- Terraform released v1.14.8.
- Oracle AI Database@Azure added a new Oracle Autonomous AI Database - Dedicated option.
- Planetary Computer highlighted geospatial intelligence for the next era of energy decision-making.
