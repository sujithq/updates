# Weekly Digest: Azure, GitHub, and Developer Ecosystem  
**June 22 – June 29, 2026**

## Executive summary

This week’s announcements were led by agentic AI, with Microsoft and GitHub pushing deeper into enterprise Copilot capabilities, agent evaluation, memory safety, and operational tooling for multi-agent systems. GitHub also shipped a strong set of platform improvements for enterprise development teams, including better reporting, more flexible Actions execution, tighter runner controls, and expanded Copilot capabilities. On Azure, the biggest themes were production-readiness for AI platforms, secure infrastructure at scale, and practical platform updates spanning App Service, Container Registry, observability, and migration planning. Across the broader ecosystem, developers also got meaningful updates in VS Code, Azure developer tooling, Kubernetes UX, and local AI on Windows.

## Copilot & Agents

Enterprise agent adoption is becoming more practical, with announcements focused on employee upskilling, faster agent onboarding, and the evolving role of AI collaboration at work.

- Learning Agent is now generally available for personalized AI upskilling across employees.
- Agent 365 Skills aims to let teams bring agents into Microsoft Agent 365 in minutes.
- “(Co)work in Progress” explores how AI-assisted work is continuing to evolve in Microsoft 365 Copilot.

## GitHub

GitHub’s updates this week centered on enterprise controls, Copilot quality and efficiency, and developer productivity improvements across desktop, Actions, runners, and issue management.

- Enterprise and organization reports now track total merges by adoption phase.
- MAI-Code-1-Flash is now available for Copilot Business and Copilot Enterprise.

- GitHub-hosted runners now offer more control.
- Actions steps can now be run in parallel.
- Red Hat Enterprise Linux runner images entered public preview.
- npm added preventive account protection for high-impact accounts.
