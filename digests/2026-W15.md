# Weekly Digest: March 30 – April 06, 2026

## Executive summary

This week’s announcements centered on three clear themes: AI agents moving from experimentation into enterprise implementation, GitHub and developer tools getting more agent-centric, and Azure continuing to harden its platform with governance, security, and lifecycle updates. Microsoft Foundry, Azure Functions, Visual Studio, and GitHub all introduced capabilities that make it easier to build, run, govern, and observe agents across local, cloud, and enterprise network boundaries. On the platform side, Azure shipped meaningful infrastructure and service updates, including new GA milestones in networking, storage access, eventing, and scheduling. Security and migration guidance also stood out, with strong emphasis on AI risk, supply chain protection, Sentinel expansion, and cost-aware modernization.

## Copilot & Agents

This was a smaller but strategically important week for Copilot, focused on improving how people create, collaborate, and reason with AI inside Microsoft 365. The updates point to richer notebook workflows and more capable research experiences powered by multiple models.

- Copilot Notebooks adds enhancements for content creation, collaboration, and learning workflows.
- What’s New in Microsoft 365 Copilot | March 2026 rounds up the latest product improvements across the suite.
- Researcher introduces multi-model intelligence to broaden and strengthen research task execution.

## GitHub

GitHub’s week was heavily focused on enterprise Copilot controls, agent workflows, and platform usability. The biggest story is that Copilot is becoming more governable and more operationally integrated across organizations, while core developer surfaces like Actions, Issues, and Visual Studio continue to improve.

- Organizations can now apply runner controls for Copilot cloud agent.
- Organizations can now configure firewall settings for Copilot cloud agent.

- GitHub Copilot in Visual Studio gets its March update.
- Improved search for GitHub Issues is now generally available.
- Copilot CLI adds `/fleet` to run multiple agents at once.
- GitHub highlighted broader open source supply chain security work across the platform.

## Data & AI

Data and AI announcements were dominated by Microsoft Foundry, with new models, agent capabilities, evaluation guidance, and enterprise deployment patterns. The key trend is a maturing stack for building production-grade AI systems: better models, better memory, better voice interfaces, and better operational guidance for RAG and agent architectures.

- Microsoft Foundry introduces MAI-Transcribe-1, MAI-Voice-1, and MAI-Image-2.
- Voice-native agents in Microsoft Foundry enter public preview.
- Foundry adds user-scoped persistent memory for adaptive, personalized agents.
- Foundry adds new models including NVIDIA Nemotron-3-Super-120B-A12B, IBM Granite-4.0-1b-Speech, and Sarvam-105B.
- Foundry agents and custom engine agents can now be discussed in the context of operating through the corporate firewall.
- Azure AI Search guidance explains why vector drift can degrade RAG accuracy after deployment.
- New guidance covers evaluating LLM truthfulness and quality in RAG systems.
- Microsoft shares a model upgrade and migration strategy for Foundry.
- TSGen introduces AI-built automated troubleshooting guide generation at scale.
- A new customer story shows how Veris AI and Lume Security built a self-improving AI agent with Microsoft Foundry.
