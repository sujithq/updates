# Weekly Digest: June 15–22, 2026

## Executive summary

This week’s announcements centered on AI-assisted development, agent engineering, and the operational controls needed to run those systems safely at scale. GitHub and VS Code both shipped meaningful Copilot improvements, while Microsoft Foundry expanded its toolkit for evaluation, benchmarking, trace selection, and enterprise RL. On the platform side, Azure highlighted observability automation, migration tooling, networking enhancements, and incremental AKS, Cosmos DB, and Databricks updates. Security remained a major theme, with new research on AI agent risk, software supply chain attacks, unified SOC workflows, and stronger cloud and container defenses.

## Copilot & Agents

The lone item in this category underscored a broader industry pattern: agentic work is still evolving from experimentation into practical collaboration models. The emphasis is on how people and copilots actually work together, not just on model capability.

- Microsoft 365 Copilot’s “(Co)work in Progress” explores the evolving nature of human-agent collaboration.

## GitHub

GitHub’s week was packed with Copilot refinements, Actions security controls, and quality-of-life improvements for day-to-day repository work. The biggest themes were better context handling for AI coding, stronger workflow safety, and more visibility into usage and automation.

- Copilot usage metrics API now includes AI credits consumed per user.
- MAI-Code-1-Flash is now available on more Copilot surfaces.
- Copilot code review adds AGENTS.md support and UI improvements.
- Duplicate issue detection enters public preview, alongside issue fields MCP support for GitHub Issues.
- Copilot-authored pull requests are now included in author searches.
- Repository switcher is now generally available in global navigation.
- GitHub added custom-image chaining support for Actions builds.
- GitHub hardened `pull_request_target` checkout defaults for safer Actions behavior.
- New controls let teams govern who and what can trigger GitHub Actions workflows.
- GitHub detailed improvements to Copilot context handling and model routing for better token efficiency.

## Data & AI

Microsoft Foundry dominated this category with a strong focus on evaluation, benchmarking, production trace quality, and agent-building patterns for enterprise AI. Azure Databricks and Cosmos DB also advanced integration and resilience with OneLake support, immutable backups, and Logic Apps connectivity.

- Microsoft Foundry introduced benchmark capabilities in preview for standardized model and agent quality checks.
- Foundry added auto-generated rubric evaluators for context-aware AI agent evaluation.
- Foundry detailed intelligent sampling for selecting better production traces.
- OpenEnv and Foundry were positioned together for outcome-driven enterprise reinforcement learning.
- Azure AI Foundry published supported cross-region model connectivity patterns and tradeoffs.
- Foundry showed how to build an automated SLA risk agent with Routines.
- Foundry introduced USR-8 for scoring user simulators.
- Microsoft launched a guided tour of the new Foundry Labs.
- Azure Cosmos DB Backup public preview adds immutable backups and long-term retention.
- Azure Cosmos DB built-in connector for Logic Apps Standard reached general availability.

## Developer Tools

Developer tooling updates focused on faster inner loops, better AI integration, and more customization across VS Code, Visual Studio, terminals, and .NET build diagnostics. A notable thread was reducing token waste and improving practical AI ergonomics for real-world engineering workflows.

- Visual Studio Code 1.126 shipped, following the 1.125 release.
- VS Code now supports bringing your own language model key.
- VS Code shared work on improving token efficiency for GitHub Copilot.
- VS Code published lessons from 50,000 runs of a lightweight eval.

- Visual Studio published new personalization options for the IDE.
- Microsoft explored failure modes where agent extensions conflict with each other.
- Microsoft also published guidance on avoiding overloaded skills in agent design.

## Azure Platform

Azure platform content this week emphasized modernization paths for integration workloads, secure tool access patterns, and skilling for the agentic era. The updates are most relevant to teams managing Logic Apps, API mediation, and developer enablement.

- Logic Apps shared the migration path from in-proc to out-of-proc hosting on the road to .NET 10 support.
- APIM MCP Gateway was positioned as a way to control tool access.
- Azure Arc published its May 2026 server forum update.
- ArchAngel focused on preparing the next generation of developers for agentic transformation.

## Azure Infrastructure

Infrastructure announcements concentrated on networking evolution, large-scale connectivity, and high-performance compute for AI and graphics-intensive workloads. The most actionable changes were around NAT Gateway, Azure Firewall migration, and VNet connectivity design.

- ICMP support for Azure StandardV2 NAT Gateway was announced.
- Azure published guidance for migrating Azure Firewall explicit proxy deployments.
- Azure outlined migration from MSEE hairpin routing to AVNM mesh for large-scale VNet-to-VNet connectivity.
- Azure Virtual Desktop showcased scaling high-performance CAD with NVIDIA RTX PRO 6000.
- Azure claimed a new extreme-scale LLM training benchmark performance record.
- Network security perimeter for Azure Service Bus is now also available in Azure Government regions.

## Containers & Kubernetes

AKS had a relatively quiet week, with the main signal coming through release notes. For platform teams, this is a reminder to review incremental service changes rather than expect one major headline each week.

- AKS published the 2026-05-29 release notes.

## Security

Security news was especially strong this week, spanning AI agent exploitation, npm supply chain compromise analysis, SOC modernization, and cloud-native defense patterns. The throughline was clear: AI increases both attack surface and defender expectations, pushing teams toward more unified, automated security operations.

- Microsoft detailed AutoJack, showing how a single page can RCE the host running an AI agent.
- Microsoft analyzed the Mastra npm supply chain compromise attributed to Sapphire Sleet.
- New threat research covered Crypto Clipper’s Tor-based control and worm-like propagation.
- Microsoft argued for advancing security “at AI speed” beyond traditional benchmarks.
- Sentinel announced new additions to normalization and ASIM.
- Sentinel introduced a public preview Agent Identities Asset Connector.
- Microsoft highlighted a unified security operations experience in Defender.
- Defender for Cloud examined end-to-end container security from code to runtime.
- Azure Network Security published a DMZ-first, traffic-flow-based landing zone architecture.
- Microsoft shared one year of Defender email security benchmarking insights.

This category focused on cost planning, modernization at scale, observability automation, and migration execution. The standout theme was using AI and Copilot-adjacent tooling to accelerate troubleshooting, code assessment, and cloud transition work.

- Azure Observability introduced the Copilot Observability Agent for faster AKS troubleshooting.

- Dynamic thresholds for Log search alerts make anomaly detection easier.

- Azure Migration shared three lessons that make or break AWS-to-Azure migrations.
- Microsoft compared options to migrate SQL Server databases.
- Memory-optimized table variables were examined from a performance perspective.

## Azure Updates

Azure Updates this week reinforced themes already visible across the blogs: modernization, Databricks-OneLake integration, improved networking, and observability improvements. Most items were either previews aimed at platform builders or GA milestones for production teams.

- Azure Migrate–GitHub Copilot Modernization integration for at-scale code assessments entered public preview.
- Azure Databricks native read access to Microsoft OneLake reached general availability.
- Azure Databricks native storage of data in Microsoft OneLake entered public preview.
- ICMP support for Azure Standard V2 NAT Gateway reached general availability.
