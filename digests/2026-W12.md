# Weekly Digest: March 15–22, 2026

## Executive summary

This week’s news was dominated by Microsoft’s push to make agentic AI production-ready: Microsoft Foundry reached key GA milestones for agent services, evaluations, monitoring, and tracing, while voice-native and real-time agent experiences expanded in preview. GitHub continued tightening the Copilot developer loop with faster coding-agent startup, better session visibility, richer metrics, and improved traceability from commits back to agent logs. Across Azure, platform and infrastructure updates focused on inference at scale, resiliency, cost optimization, and migration planning, while security announcements centered on protecting AI systems, expanding Sentinel capabilities, and improving detection pipelines. The broader developer toolchain also kept evolving with major VS Code releases, AI-assisted SQL tooling, and local-to-hosted workflows for building and testing agents.

## Copilot & Agents

Microsoft 365 and Security Copilot updates focused on operationalizing agents safely inside enterprise workflows, while expanding the surfaces where Copilot-derived intelligence appears. The theme was practical adoption: pricing, governance, app integration, and security-specific automation.

- People Skills expands skills intelligence to more users and Microsoft 365 surfaces.
- Security Copilot agents are positioned as a response to alert overload in security and IT operations.
- The Microsoft Agent 365 AMA highlighted current direction and common questions around Agent 365.
- Secret Finder targets discovery of real credentials missed by traditional tools.
- Microsoft is urging customers to lock in current pricing for Microsoft 365 Copilot business bundles.
- New Microsoft 365 Copilot capabilities let agents bring apps into the flow of work with IT control.

## GitHub

GitHub’s announcements centered on making Copilot coding agents faster, more observable, and easier to govern in team workflows. Beyond Copilot, GitHub shipped incremental improvements across Actions, Codespaces, mobile, and pull request review ergonomics.

- Copilot usage metrics now attribute auto model selection to the actual model used.
- Copilot coding agent commits can now be traced back to their session logs.
- Raycast can now monitor Copilot coding agent logs live.
- GitHub added more visibility into Copilot coding agent sessions.
- Copilot coding agent now starts work 50% faster.

- GitHub Mobile for Android got a smoother navigation experience.

This was a major week for Microsoft Foundry and adjacent Azure AI services, with GA announcements for core enterprise agent capabilities and a steady flow of new models. Databricks, Azure SQL, and Cosmos DB also advanced developer workflows for vector search, CDC, mirroring, and AI-assisted database tooling.

- Microsoft Foundry made evaluations, monitoring, and tracing generally available.

- Voice-native agents in Microsoft Foundry entered public preview.
- Microsoft published a practical guide to evaluating AI agents with Foundry.

- Cosmos DB added a Cursor plugin and brought Fabric mirroring with private networks to GA.

- Defender updates expanded protections for multi-cloud, containers, AI models, and AI-era threats.
- Microsoft published observability guidance for AI systems to improve proactive risk detection.
- Purview announced new Fabric-focused innovations to support safe AI transformation.

## Governance & Operations

- A migration case study covered porting C++ from IBM Power to x86 with help from GitHub Copilot.

For developers and architects, the most important signals were AI-assisted data tooling, SQL MCP, identity improvements, and planning for upcoming service retirements.
