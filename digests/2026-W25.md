# Weekly Digest: June 08–15, 2026

## Executive summary

This week’s announcements centered on agentic development moving from demos into platform features: GitHub expanded agentic workflows and Copilot controls, while Microsoft pushed deeper agent support across Foundry, Azure DevOps, and Microsoft 365 Copilot. On Azure, the biggest themes were operational maturity and developer productivity, with notable updates for App Service, Functions, Logic Apps, Azure Monitor, and Azure DevOps. Data and AI news emphasized model breadth, browser/tool automation, and practical guidance for capacity, orchestration, and hosted agents. Azure Updates also carried an unusually important set of retirements and GA notices that teams should review for Batch, storage, networking, SQL, PostgreSQL, disks, and Redis planning.

## Copilot & Agents

Microsoft 365’s Copilot and agent announcements focused on new domain-specific agents, broader notebook updates, and continued platform iteration across Copilot and Agent 365. The theme is steady productization of AI assistants for knowledge work and workflow automation.

- Word introduced Legal Agent in Frontier for legal-document workflows.
- Microsoft 365 Copilot Notebooks received both May and June 2026 feature updates.
- Anthropic Claude Fable 5 is now available in Microsoft 365 Copilot.
- Microsoft 365 Copilot published its May 2026 roundup of product updates.
- Agent 365 published its May 2026 release roundup.

## GitHub

GitHub’s updates were dense and practical, with major attention on Copilot CLI, code review controls, Actions runner governance, and public preview for agentic workflows. Security and platform trust also stood out through secret scanning quality improvements and tighter workflow approval behavior.

- GitHub Agentic Workflows entered public preview.
- Agentic workflows no longer require a personal access token.
- GitHub Copilot code review added new configurations and controls.
- GitHub Copilot CLI added centralized configuration via `/settings`.
- GitHub detailed how Copilot CLI became more selective about delegation.
- GitHub added language-server-based code intelligence to Copilot CLI.
- GitHub showed how to move from one-off prompts to custom-agent workflows in Copilot CLI.
- GitHub Actions published the minimum-version enforcement timeline for self-hosted runners.
- Bot-created pull requests can now run workflows if approved.
- GitHub Enterprise Server 3.21 reached general availability.

## Data & AI

This week’s Data & AI news was heavily oriented toward Foundry: new models, new tools, hosted agents, and practical implementation guidance. Azure Databricks and AI integration patterns also featured prominently, especially around capacity planning and “better together” data architectures.

- Microsoft Foundry added new MAI models across text, image, voice, and speech.
- Foundry added Command A+ (W4A4), Chandra OCR 2, and GLM-OCR.
- Foundry introduced a new browser automation tool with Toolboxes.
- Azure AI Foundry shared guidance on measuring token impact of MCP tool invocation.
- Azure AI Foundry outlined transitions from Azure Language features to Foundry models.
- Foundry hosted agents were positioned for driving stop-and-go business processes to closure.
- Azure AI Foundry published an Online Meeting Effectiveness Index (OMEI).
- Guidance was published for connecting Claude clients through Azure API Management to Claude models in Foundry.
- Azure Databricks teams published capacity mitigation and planning guidance.
- Microsoft Fabric and Azure Databricks were highlighted as a complementary architecture story.

## Developer Tools

Developer tooling news spanned IDE updates, agent frameworks, Azure DevOps automation, container troubleshooting, and .NET platform releases. The clear trend is tighter AI assistance across the software lifecycle, from coding and review to testing, deployment, and modernization.

- Visual Studio Code 1.125 shipped, following 1.124 updates.
- .NET 11 Preview 5 is now available.
- .NET and .NET Framework June 2026 servicing releases were published.
- Azure DevOps Server received June patches.
- Azure DevOps introduced Copilot Autofix for GitHub Advanced Security.
- Azure DevOps added Copilot Code Reviews for Azure Repos.
- Visual Studio now supports reviewing pull requests without leaving the IDE.
- Aspire and Microsoft Agent Framework were used to demonstrate distributed multi-agent systems.
- Microsoft developers recapped Build 2026 launches, vision, and top sessions.
- PowerToys 0.100 shipped with a new Shortcut Guide and Command Palette improvements.

## Azure Platform

Azure platform announcements focused on application hosting, integration, and operational tooling, with strong momentum behind Logic Apps, App Service, Functions, API Management, and long-running ARM operations. Many of the updates help teams make cloud apps easier to build, debug, version, and automate.

- Azure Functions published its Build 2026 update.
- Azure App Service shared its Build 2026 feature roundup.
- App Service for Linux added faster startup debugging with Startup Logs.
- Logic Apps Standard introduced dynamic connection properties.
- Logic Apps Standard introduced a C# SDK.
- Azure API Management added guidance to productize, observe, version, and automate MCP servers.
- Azure shared how ARM tracks work that takes hours.
- Azure published operational guidance for running a geo-replicated ACR.
- Logic Apps Aviators released the June 2026 newsletter.
- ArchAngel highlighted developer skilling for the agentic transformation.

## Azure Infrastructure

Infrastructure news highlighted AI-ready file data pipelines, agentic compute, specialized GPU workloads, and networking security posture. The strongest theme was turning enterprise data and infrastructure into usable foundations for AI systems.

- Azure NetApp Files was positioned as a bridge between enterprise file data, Foundry, and Microsoft 365 Copilot.
- Azure NetApp Files and OneLake were presented as an AI-ready data foundation.
- Azure NetApp Files object REST API was highlighted for AI-powered knowledge pipelines.
- Azure Compute previewed ACI Sandboxes as a future direction for agentic compute.
- Azure announced real-time simulation with Ansys Discovery on AMD Radeon V710 GPUs.
- Ginkgo Bioworks and Microsoft Discovery showcased agentic AI for biological discovery.
- Network Security Perimeter for Azure Service Bus reached GA and expanded to Azure Government regions.

## Containers & Kubernetes

The container ecosystem was quieter this week, with the primary notable item being upstream Kubernetes storage community activity. For platform teams, it’s a reminder that storage remains one of the most strategic parts of Kubernetes operations.

- Kubernetes published a spotlight on SIG Storage.

## Security

Security announcements focused on AI-era investigation, evaluation, and attacker behavior, alongside deeper Azure network access auditing. The common thread was improving visibility and trust in both AI systems and privileged access.

- Azure published a deep dive into Bastion session recording.
- Microsoft Security introduced ASSERT to turn specs into evals for any agent.
- Microsoft detailed how to reconstruct AI activity in investigations.
- Microsoft warned that threat actors are using AI brands as social-engineering bait.

## Governance & Operations

Operations and governance updates were strong this week, especially in Azure Monitor and FinOps. Microsoft continues to connect metrics, traces, logs, and SLIs while also giving cost-conscious teams better levers for commitment planning and SQL migration.

- Azure Monitor SLIs reached general availability.
- Azure Monitor Metrics Export reached general availability.
- Azure Monitor previewed large-scale collection of Azure Resource Platform Logs with DCRs.
- Azure Monitor added exemplars to connect metrics to traces.
- Azure Observability emphasized “any source, any destination” readiness for the AI era.
- Azure published modern VM monitoring guidance powered by OpenTelemetry.
- FinOps guidance covered right-sizing Azure Savings Plans one hour at a time.
- Microsoft published options to migrate SQL Server databases.

## Azure Updates

Azure Updates carried several high-impact lifecycle notices alongside a healthy set of GA and preview releases. Teams should pay particular attention to retirements affecting Azure Batch pools, GPv1 storage, NAT rule v1, Synapse Link for Cosmos DB NoSQL, and the Linux Azure VPN Client preview.

- Multiple Azure Batch pool VM families are being retired, including Av2, F/Fs/Fsv2, G/Gs, Lsv2, D/Ds/Dv2/Dsv2, and Ls series.
- Creation of GPv1 and Legacy Blob storage accounts is being retired.
- Azure Load Balancer Inbound NAT rule version 1 for VMSS is being retired.
- Azure Synapse Link for Azure Cosmos DB NoSQL is being retired.
- Azure VPN Client for Linux (Preview) will retire on August 31, 2026.
- SQL MCP Server reached general availability.
- Microsoft Entra server principals on Azure SQL Database reached general availability.
- PostgreSQL Hub for Azure Developers reached general availability.
- Premium SSD v2 now supports non-zonal Azure virtual machines.
- Azure Managed Redis preview now supports Entra ID-based RBAC for data management.

## Ecosystem

The broader ecosystem updates were small but notable, covering Azure Linux, production-readiness guidance for agents, and Terraform maintenance. These items matter most to teams standardizing their platform foundation and delivery practices.

- Azure Linux 4.0 was announced in public preview.
- Azure Architecture announced the Path to Production for Agents webinar series.
- Terraform released v1.15.6.
